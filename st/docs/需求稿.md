# AceZero「命运宏图」系统需求稿 / 规则书 v0.1

> 目标：把现有 AceZero 从散装 AIRP + 游戏插件，收束成一个 **24 节点的命运节点系统**。
> 它不是具象地图、商店、战斗房间，而是一个抽象的“命运编制系统”。
> 玩家不是在传统地图上走格子，而是在命运结构里选择路线、投入点数、触发角色、打牌、成长、避险、收束结局。

---

# 0. 当前企划总方向

## 0.1 项目方向

当前采用：

## **收束型方向一**
即：

- 保留 AceZero 世界观、人设、AIRP 气质。
- 不彻底转重游戏化。
- 不彻底转 VN。
- 用 24 节点命运宏图作为主框架，把剧情、角色、游戏系统收束起来。
- 允许它不是完美开放世界，但要能闭环、能跑、能对电波。

核心目标不是做无限开放沙盒，而是做一个：

> **有世界观魂、有角色触发、有赌局玩法、有路线选择、有结局收束的 AIRP + Roguelike 命运节点系统。**

---

# 1. 命运宏图总体结构

## 1.1 节点规模

总节点位：

```text
16 Nodes
```

结构：

```text
Node 1-3：初章固定引导
Node 4-15：主体命运宏图
Node 16：终局 / 收束节点
```

有效自由区大约为：

```text
12 Nodes
```

也就是 Node4-15。

---

## 1.2 节点不是具象地图

命运宏图不是：

- 传统地图
- 商店房间
- 战斗房间
- 真实地理路线
- 剧情章节硬线

而是：

## **抽象命运路线图**

它表示玩家这轮命运正在往什么方向偏移。

真实地理层级仍然作为独立变量存在，例如：

- 上庭
- 中市
- 下街
- 底锈

但地理层不直接等于 lane。

# 2. 四条 Lane

## 2.1 Lane 定义

当前命运宏图有四条 lane：

```text
上 lane
中上 lane
中下 lane
下 lane
```

颜色可以先沿用当前 UI 的白 / 蓝 / 橙 / 红，但颜色不是固定语义，后续可以改。

---

## 2.2 四条 Lane 当前功能定位

当前建议定义：

| Lane | 本质 | 倾向 |
|---|---|---|
| 上 lane | Rest 种田 | 恢复、经营、稳定、固定收益 |
| 中上 lane | Asset 交易 | 成长、升级、契约、资产构筑 |
| 中下 lane | Vision 再探再报 | 探路、跳线、规避、替换固定点 |
| 下 lane | Combat 塔塔开 | 打牌、冲突、赌博、高风险高收益 |

这不是硬性说“上 lane 只能 Rest”，而是说：

> 这条 lane 的限定点数和事件风格更偏向该类型。

---

## 2.3 Lane 的作用

Lane 不是传统主线。

它的作用是：

1. 给玩家“有方向的自由”。
2. 区分纯乱走和有规划的路线。
3. 给角色刷新、资源产出、节点奖励提供偏向。
4. 给系统一个可控的路线语法。
5. 允许玩家通过 Vision 跳线打破路线限制。

# 4. 点数来源

点数来源分三类：

## 4.1 节点限定点数

每个节点会给限定点数。

特点：

- 进入当前节点时获得。
- 如果进入下一个节点前没有使用，则消失。
- 限定点数用于鼓励玩家在当前节点行动。
- 消耗时优先消耗限定点数，再消耗储备点数。

Lane 对限定点数有偏向：

| Lane | 限定点数偏向 |
|---|---|
| 上 lane | Rest 多 |
| 中上 lane | Asset 多 |
| 中下 lane | Vision 多 |
| 下 lane | Combat 多 |

---

## 4.2 Combat 战斗收益

Combat 点投入后，会触发赌局 / 小游戏 / 德州 / DLC日麻。

根据战况，产出其他点数，例如：

- Vision
- Asset
- Rest
- Combat

产出数量随投入等级变化。

输了会有惩罚，且投入越高惩罚越大。

---

## 4.3 固定营收

新增固定营收机制。

每个点数都有一个固定营收进度。

建议字段：

```js
income_rate: {
  combat: 0.2,
  rest: 0.2,
  asset: 0.2,
  vision: 0.2
}
```

初始每个点数的营收率为：

```text
0.2 / node
```

上限：

```text
1.5 / node
```

每推进一个节点，按营收率增加对应点数的进度。

当进度满 1 时，转化为 1 个储备点数。

---

# 5. 相位投入机制

每个节点有 4 个 phase。

每个 phase 可以投入一种点数，投入等级最多 3。

```text
投入 1 点 = 一级效果
投入 2 点 = 二级效果
投入 3 点 = 三级效果
```

建议数据结构从旧的：

```js
phase_slots[index] = { key, source }
```

升级为：

```js
phase_slots[index] = {
  key: 'combat' | 'rest' | 'asset' | 'vision',
  amount: 1 | 2 | 3,
  sources: ['limited', 'reserve']
}
```

# 6. Combat 点数规则

## 6.1 Combat 本质

Combat 是主动进入冲突的点数。

它触发：

- 小游戏
- 德州
- 精英德州
- Boss 德州
- DLC 日麻局

---

## 6.2 Combat 投入等级

### 投入 1 点

触发：

- 小游戏
- 低端德州局

赌局规模：

```text
当前 funds / 资产 的 10%
```

回报点数：

```text
0-2
```

---

### 投入 2 点

触发：

- 精英德州局
- 初级日麻局，前提是玩家明确加入日麻 DLC

赌局规模：

```text
当前 funds / 资产 的 33%
```

回报点数：

```text
0-5
```

---

### 投入 3 点

触发：

- Boss 德州局
- Boss 日麻局，前提是玩家明确加入日麻 DLC

赌局规模：

```text
当前 funds / 资产 的 70%
```

回报点数：

```text
0-9
```

---

## 6.3 Combat 备注

赌局规模只参考：

```text
funds + 正资产
```

不把负债算作可用筹码。

但剧情上可以表现为玩家通过杠杆、借贷、抵押扩大风险。

---

# 7. Rest 点数规则

## 7.1 Rest 本质

Rest 是休整、回复、经营、占点、固定收益的点数。

它不是单纯睡觉，而是：

- 恢复 Mana
- 稳定状态
- 控制节点
- 增加固定营收
- 给命运宏图染色

---

## 7.2 Rest 投入等级

### 投入 1 点

效果：

```text
回复最大 Mana 的 25%
```

可选追加：

```text
选择一个点数染色，并消耗 1 个对应点数，使该点数固定营收 +0.1
```

---

### 投入 2 点

效果：

```text
回复最大 Mana 的 66%
```

可选追加：

```text
选择一个点数染色，并消耗 1 个对应点数，使该点数固定营收 +0.25
```

---

### 投入 3 点

效果：

```text
回复最大 Mana 的 100%
```

可选追加：

```text
选择一个点数染色，并消耗 1 个对应点数，使该点数固定营收 +0.4
```

---

## 7.3 节点染色 / 控制

Rest 可以把当前节点变成玩家控制点。

如果选择点数染色：

```text
当前节点被染成对应点数类型。
```

如果不选择点数染色：

```text
当前节点高亮为普通固定收益点。
```

---

## 7.4 染色收益爆发

当玩家进入某条 lane 时，如果该 lane 上已有被染色的节点，则有概率爆出对应点数。

收益：

```text
2-3 点对应点数
```

概率与该 lane 上同类染色节点数量相关。

例如：

```text
中下 lane 上有多个 Vision 染色节点，则玩家进入中下 lane 时更容易爆出 2-3 Vision 点。
```

---

# 8. Asset 点数规则

## 8.1 Asset 本质

Asset 是命运资产，不等于具体金钱。

它代表：

- 契约权限
- 成长权限
- 技能升级
- Mana 上限提升
- 学习技能

不要和 funds / 资产 / 负债混淆。

funds 是剧情经济资产。

Asset 点数是命运宏图系统资源。

---

## 8.2 Asset 投入等级

### 投入 1 点：一级契约互动

可选择：

- 升级技能，最高到 2 级
- Mana 上限 +3
- 学习初级技能

---

### 投入 2 点：二级契约互动

可选择：

- 升级技能，最高到 3 级
- Mana 上限 +7
- 学习中级技能

---

### 投入 3 点：三级契约互动

可选择：

- 升级技能，最高到 4 级
- Mana 上限 +12
- 学习高级技能

---

# 9. Vision 点数规则

## 9.1 Vision 本质

Vision 是探路、绕线、替换固定相位、命运干涉的点数。

它不再是不可控 Event。

它是玩家主动掌控命运路径的权限。

---

## 9.2 默认视野

玩家默认视野：

```text
1 node
```

即默认只能看清前方 1 个节点范围。

---

## 9.3 Vision 投入等级

### 投入 1 点：探路

效果：

```text
当前视野 +2
```

用于提前看见更远的节点信息。

---

### 投入 2 点：替换固定点位

效果：

```text
获得 1 次固定相位干涉机会。
```

规则：

- 消耗 2 Vision。
- 不需要预先绑定目标 fixed phase。
- 之后每次抵达 fixed phase 前，底部 phase 条弹出选择。
- 选择 KEEP 时，保持原固定点，不消耗这次机会。
- 选择替换点数时，抵消原 fixed phase 1 次。
- 替换后生效为一个玩家选择的普通点数相位。
- 如果一直不用，这次机会可以跨节点保留。
- 重要剧情固定点位需要提示玩家确认。
- 某些关键剧情可以被标记为不可替换，或替换后只降级为前置信号。

用途：

```text
绕过不想打的陷阱 / 处刑 / 损失事件。
```

---

### 投入 3 点：跳线 / 跃迁

效果：

```text
到达对应 phase 时，进入跳线模式。
```

跳线模式：

- 无视当前路线连接。
- 可以自由选择下一节点。
- 可以跨 lane。
- 不能跳到已禁止或未开放节点。
- 不建议允许跳过终局硬锁。

---

# 10. 危机值系统

新增全局危机值：

```text
crisis: 0-100
```

危机值表示当前局势危险程度。

用于：

- 高危角色触发
- 管理局介入
- 教廷介入
- Trixie / Kako / Eulalia 等稀有角色触发

危机值来源后续可细化。

建议来源包括：

- 高级 Combat
- Combat 失败
- 负债增长
- 替换固定剧情
- 跳线
- 连续使用 Vision
- Boss 局
- 重要事件失败

# 12. 角色触发状态机

建议每个角色有以下状态：

```js
characterEncounter: {
  SIA: {
    status: 'locked' | 'eligible' | 'queued' | 'pre_signal' | 'first_meet' | 'introduced',
    firstMeetDone: false,
    preSignalDone: false,
    cooldown: 0
  }
}
```

状态含义：

| 状态 | 含义 |
|---|---|
| locked | 尚未满足条件 |
| eligible | 条件满足，可进入候选池 |
| queued | 已排队等待投放 |
| pre_signal | 已刷前置信号 |
| first_meet | 正式初见待执行 |
| introduced | 已正式初见 |

---

# 13. 初见容量限制

由于有效节点只有约 12 个，平均一轮目标触发：

```text
4-5 个角色
```

不是一轮把 8 个角色全刷完。

因此必须有容量限制。

建议规则：

```text
每个 node 最多 1 个正式初见。
每个 node 最多 1 个前置信号。
正式初见后至少冷却 1 个 node，再允许下一个正式初见。
```

如果多个角色同时满足条件，进入初见队列。

队列排序可按：

1. 是否已刷前置信号
2. 危机值需求
3. 当前 lane 权重
4. 当前地理层级
5. 角色稀有度
6. 是否已有前置角色触发

# 19. 系统一句话定义

## 命运宏图系统是：

> 一个 24 节点、4 lane、4 phase、4 resource 的抽象命运编制系统。
> 玩家通过投入 Combat / Rest / Asset / Vision 改写当前节点的相位，获得收益、成长、视野、跳线、固定营收与角色接触机会。
> 角色不是固定章节出场，而是根据玩家行为、危机、路线、地理与前置事件，被投放到玩家附近 1-2 个节点内。
> 最终目标是在保留 AIRP 自由感的同时，用资源和节点结构让 AceZero 可以闭环运行。
