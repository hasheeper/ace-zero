<variable_rule>
ACE0 MVU 变量规则
仅输出本轮真实变化；禁止整段覆写。

——【相位定义】——
“相位”是理解节点内情节推进用的单位，不是世界时间。
- 一个节点最多四个相位：一段 / 二段 / 三段 / 四段
- 相位表示“当前节点里的这一小段戏 / 当前正在写的这一步情节”
- 你可以把它理解为：同一节点内部，剧情从开场、试探、推进、收束，分成最多四步来写
- 它不表示推进 DAY，也不表示推进 MORNING / NOON / AFTERNOON / NIGHT
- 只有当前这一小段情节已经自然收束、确实该进入下一步时，才写 `phase_advance`

允许更新的路径
- `/hero/funds`
- `/hero/assets`
- `/hero/debt`
- `/hero/majorDebt`
- `/world/current_time/day`
- `/world/current_time/phase`
- `/world/location/layer`
- `/world/location/site`
- `/world/act/route_history`
- `/world/act/phase_advance`
- `/world/act/eventTree/nodeGoals`
- `/world/act/eventTree/phaseWindow`
- `/world/act/phasePlanLock`
- `/world/act/reserve/combat`
- `/world/act/reserve/rest`
- `/world/act/reserve/asset`
- `/world/act/reserve/vision`
- `/world/act/pendingResolutions/<index>/status`
- `/world/act/pendingResolutions/<index>/outcome`
- `/hero/cast/<角色名>/present`
- `/hero/cast/<角色名>/inParty`
- `/hero/roster/<角色名>/level`
- `/hero/roster/<角色名>/mana`
- `/hero/roster/<角色名>/maxMana`
- `/hero/relationship/<角色名>/entanglement`
- `/hero/relationship/<角色名>/exclusive`

操作约定
- `delta`: 仅用于数字增减。适用：`/hero/funds`、`/hero/assets`、`/hero/debt`、`/hero/majorDebt`、`/hero/relationship/*`
- `replace`: 仅用于直接写结果值。适用：时间、地点、ACT 路由/推进、布尔值、level、mana、maxMana
- `add`: 当前不使用。除非规则另行明确声明，否则不要追加数组项
- 不使用整段对象覆写，不新增未声明路径
- 关系第二字段统一写 `/exclusive`，无后续路径

字段约束:
  hero.funds:
    type: number
    check:
      - 仅用 `delta`
      - 单位统一为金弗（可带 2 位小数）
      - 正数=收入，负数=支出
      - 最多保留 2 位小数
      - 不重复记账
      - 无金额变化则不写
  hero.assets:
    type: number
    check:
      - 仅用 `delta`
      - 单位统一为金弗（可带 2 位小数）
      - 表示当前名下非现金资产总值
      - 买入资产通常 = `funds` 减少、`assets` 增加
      - 卖出资产通常 = `funds` 增加、`assets` 减少
      - 只有剧情明确发生资产取得、损毁、转移、出售、重估时才写
  hero.debt:
    type: number
    check:
      - 仅用 `delta`
      - 单位统一为金弗（可带 2 位小数）
      - 表示普通债务总额，不含主线大债
      - 借到现金通常 = `funds` 增加，同时 `debt` 增加
      - 偿还普通债务通常 = `funds` 减少，同时 `debt` 减少
      - 罚款、赊账、杂项欠款成立时可只增加 `debt`
      - 利息不由 LLM 手算；未来若有自动结息，由系统在时间推进后处理
  hero.majorDebt:
    type: number
    check:
      - 仅用 `delta`
      - 单位统一为金弗（可带 2 位小数）
      - 表示主线核心大债，独立于普通 `debt`
      - 只有剧情明确让主线大债本金增加、减少、重组、减免时才写
      - 利息不由 LLM 手算；未来若有自动结息，由系统在时间推进后处理
  hero.roster.${角色名}.level:
    type: number
    range: 0~5
    check:
      - 仅在升级/降级时更新
  hero.roster.${角色名}.mana:
    type: number
    check:
      - 仅写消耗/恢复后的结果值
  hero.roster.${角色名}.maxMana:
    type: number
    check:
      - 仅在上限变化时更新
  world.current_time.day:
    type: number
    check:
      - 仅在明确跨日时更新
      - 独立于 ACT 节点；节点推进不自动跨日
  world.current_time.phase:
    type: enum
    allowed: MORNING / NOON / AFTERNOON / NIGHT
    check:
      - 仅在明确推进世界时段时更新
      - 独立于 ACT 节点；节点推进不自动推时段
  world.location.layer:
    type: enum
    allowed: THE_COURT / THE_EXCHANGE / THE_STREET / THE_RUST
    check:
      - 仅在明确跨层时更新
      - 跨层时优先同步更新 `site`
  world.location.site:
    type: string
    check:
      - 仅在明确换场时更新
      - 写当前镜头场所
  world.act.route_history:
    type: array
    check:
      - 仅在明确选择下一节点时整体替换（追加新 node id）
      - 必须保持顺序，表示已走过的节点
      - 不要写派生字段，如 `current_node_id / dead_nodes / future_nodes`
  world.act.phase_advance:
    type: number
    range: 0~4
    check:
      - 仅在需要请求系统推进段位时写
      - 这里的“段位 / 相位” = 当前节点内部当前这一步情节，不是世界时段
      - 写 1 = 推进 1 段；写 4 = 连推 4 段
      - 只有当前相位已经写到自然收束时才写；若仍在展开 / 对话 / 试探中，就不要写
  world.act.eventTree.nodeGoals:
    type: object
    check:
      - 仅用于维护和计划当前节点与下一节点的剧情总目标
      - 只允许 `current` 与 `next`
      - `current` 必须写成 `{ "goal": "...", "tendency": "..." }`
      - `next` 必须写成 `{ "goal": "...", "tendency": "" }`；只写下一节点总目标，不写下一节点四段细分
      - 不要把 `current` / `next` 直接写成字符串
      - 仅在目标缺失、目标变化、正文现实已经偏离原计划时写
      - 使用 `replace` 整体替换 nodeGoals，不要追加数组
  world.act.eventTree.phaseWindow:
    type: object
    check:
      - 仅用于当前节点四段小目标/突发事件窗口
      - 必须写成 `{ "nodeId": "<当前节点ID>", "phases": [...] }`
      - `phases` 固定四项，每项写 `{ "index": 0~3, "goal": "...", "event": "" }`
      - 每段只写 goal/event
      - 当前节点没有 phaseWindow 时，优先规划四段小目标
      - 若正文现实偏离计划，可用 `replace` 更新当前节点 phaseWindow
  world.act.phasePlanLock:
    type: object
    check:
      - 仅由 Dashboard 的“确定规划”按钮写入
      - 剧情 LLM 不要在普通正文中写入或修改
      - 格式为 `{ nodeId, nodeIndex, locked, confirmedPhaseIndex }`
      - `locked=true` 只表示当前节点点数编排已锁定，不表示剧情目标
  world.act.reserve.${combat|rest|asset|vision}:
    type: number
    check:
      - 仅按 `<ACE0_COMBAT_SETTLEMENT>` 的 `suggestedJsonPatch` 更新
      - 只用 `delta`
      - 不自行分配点数
  world.act.pendingResolutions.${index}.status/outcome:
    type: string
    check:
      - 仅按 `<ACE0_COMBAT_SETTLEMENT>` 的 `suggestedJsonPatch` 更新
      - 对应条目只标记为 `resolved`，不删除数组项
  hero.cast.${角色名}.present:
    type: boolean
    check:
      - 仅在当前同场、同镜头、可直接动作/对话时设为 true
      - 离场、切场、未同行时设为 false
      - 只有 `activated=true` 时才有意义
  hero.cast.${角色名}.inParty:
    type: boolean
    check:
      - 仅在正式同行/入队时设为 true
      - 只有 `activated=true` 时才有意义
  hero.relationship.${角色名}.entanglement:
    type: number
    range: 0~100
    check:
      - 仅在可观察变化时更新
      - 优先用 `delta`
      - 非纠偏不要用 `replace`
      - 数值必须与事件强度匹配
  hero.relationship.${角色名}.exclusive:
    type: number
    range: 0~100
    check:
      - 仅在专属关系结构变化时更新
      - 优先用 `delta`
      - 非纠偏不要用 `replace`
      - 路径始终写 `exclusive`

输出要求:
- 末尾输出 `<UpdateVariable>`
- 无变化则不输出
- 使用 JSONPatch
- 只写最小必要改动

```xml

<JSONPatch>

  { "op": "delta", "path": "/hero/funds", "value": -2.5 },
  { "op": "delta", "path": "/hero/assets", "value": 2.5 },
  { "op": "delta", "path": "/hero/debt", "value": 10 },
  { "op": "delta", "path": "/hero/majorDebt", "value": -1000 },
  { "op": "delta", "path": "/world/act/reserve/combat", "value": 1 },

  { "op": "replace", "path": "/world/act/phase_advance", "value": 1 },
  { "op": "replace", "path": "/hero/cast/TRIXIE/present", "value": true }
]
</JSONPatch>
</UpdateVariable>
```
